# ATM Simulation (банкомат с авторизацией)

## Постановка задачи

Разработать консольное приложение, имитирующее работу банкомата. Программа должна решать следующие прикладные задачи:

- Хранение данных пользователей (логин, пароль, баланс) в текстовом файле.
- Регистрация нового пользователя с проверкой уникальности логина.
- Авторизация по логину и паролю.
- Внесение, снятие и перевод денег между пользователями.
- Сохранение всех изменений после каждой операции.
- Гарантия того, что в системе существует только один экземпляр банкомата (чтобы избежать конфликтующих операций).

Проблема: в реальном банкомате не может быть двух одновременно работающих экземпляров, управляющих одной базой клиентов. Программа имитирует это ограничение.
## Использованный паттерн: Singleton

Цель применения:
Обеспечить существование единственного экземпляра класса `Atm` во время выполнения программы.

Какую задачу решает:
- Предотвращает создание второго объекта банкомата через `new Atm()`.
- Централизует управление пользователями и операциями.
- Упрощает доступ к банкомату из любого места через `Atm.GetInstanceValue()`.
- В коде используется потокобезопасная реализация с `lock`, что полезно для потенциального многопоточного окружения.
